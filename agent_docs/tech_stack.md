# Tech Stack & Tools

## Overview
This project uses a Jamstack architecture with an offline Python ML pipeline that generates static JSON, which is then consumed by a Next.js static site generator.

## Python ML Pipeline
- **Python:** 3.11+
- **Data Processing:** pandas, numpy
- **Web Scraping:** requests, BeautifulSoup4
- **NBA Data:** nba_api (PyPI), sports_reference
- **ML Libraries:** scikit-learn, XGBoost, SHAP
- **Testing:** pytest
- **Linting/Formatting:** black (line length 100), ruff

### Setup Commands
```bash
cd pipeline
python -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
```

### Pipeline Execution
```bash
cd pipeline
bash run_pipeline.sh  # Runs full pipeline: ingest → features → models → JSON export
```

## Frontend
- **Framework:** Next.js 15.x (App Router)
- **Language:** TypeScript 5.x (strict mode)
- **Styling:** Tailwind CSS 3.x
- **UI Components:** Custom components with Tailwind utilities
- **Charts:** Recharts (React-based, accessible charting library)
- **Icons:** Lucide React (tree-shakeable icons)
- **Linting:** ESLint + Prettier

### Setup Commands

## Database
**None** — All data is pre-computed and exported as static JSON files stored in `frontend/public/data/`.

## Key JSON Outputs

All JSON files generated by the Python pipeline and placed in `frontend/public/data/`:

1. **prospects.json** — Array of all prospects with stats and predictions
2. **comparisons.json** — Object mapping prospect IDs to arrays of historical player comparisons
3. **metric_importance.json** — Array of metrics ranked by SHAP importance
4. **big_board.json** — Array of prospects ranked by predicted success

## Data Loading Pattern

```typescript
// lib/data.ts
import prospectsData from '../../public/data/prospects.json'
import comparisonsData from '../../public/data/comparisons.json'

export const getProspects = (): Prospect[] => prospectsData as Prospect[]
export const getComparisons = (prospectId: string): Comparison[] =>
  (comparisonsData as Record<string, Comparison[]>)[prospectId] ?? []
```

All data is loaded at **build time** via direct JSON imports. No runtime API calls.

## Error Handling

### Python Pipeline
```python
# All external API calls must be wrapped in try-except with caching
def fetch_with_cache(url: str, cache_path: Path) -> dict:
    """Fetch data with local cache to avoid re-scraping."""
    if cache_path.exists():
        return json.loads(cache_path.read_text())
    
    try:
        response = requests.get(url, timeout=10)
        response.raise_for_status()
        data = response.json()
        cache_path.write_text(json.dumps(data))
        return data
    except requests.RequestException as e:
        raise RuntimeError(f"Failed to fetch {url}: {e}")
```

### Frontend (TypeScript)
```typescript
// Graceful fallbacks for missing data
export const getComparisons = (prospectId: string): Comparison[] => {
  const comps = comparisonsData[prospectId]
  if (!comps) {
    console.warn(`No comparisons found for prospect: ${prospectId}`)
    return []
  }
  return comps
}
```

## Naming Conventions
- **Python files/modules:** snake_case (`data_ingestion.py`, `feature_engineering.py`)
- **TypeScript files:** kebab-case for pages/components (`prospect-card.tsx`, `big-board.tsx`)
- **React Components:** PascalCase (`ProspectCard`, `MetricChart`)
- **Functions/variables:** camelCase
- **Constants/env vars:** UPPER_SNAKE_CASE
- **TypeScript interfaces:** PascalCase with `Interface` suffix optional (`Prospect`, `MetricValue`)
